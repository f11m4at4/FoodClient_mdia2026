# TASK-03 로컬 분석 서버 통합 체크리스트

## 연결 및 계약

- [ ] 개발 PC와 테스트 폰이 같은 네트워크에 연결되었다.
- [ ] 서버가 `0.0.0.0`에 바인딩되고 지정 포트만 방화벽에서 허용되었다.
- [ ] 테스트 폰에서 사설 IP의 진단 엔드포인트에 접근할 수 있다.
- [ ] JPEG `photo` multipart 요청과 응답 필드 계약이 일치한다.
- [ ] 서버가 500자 이내 조리법과 `requestId`를 반환한다.

## 실패 및 관측성

- [ ] 정상 요청 결과가 화면에 표시되고 TTS로 재생된다.
- [ ] 지연 중 `Uploading` 상태와 취소 동작이 올바르다.
- [ ] timeout, 서버 중단, 4xx/5xx, 잘못된 JSON, 음식 미검출을 재현했다.
- [ ] 캡처 성공과 분석 실패가 UI에서 별도 메시지로 표시된다.
- [ ] logcat에서 캡처 성공, 분석 시작, 분석 실패의 순서와 원인을 확인할 수 있다.
- [ ] 서버 복구 후 재시도가 성공한다.

## 보안 및 품질 게이트

- [ ] 앱 소스, APK, BuildConfig, logcat에 OpenAI API 키가 없다.
- [ ] 사진 바이트, 조리법 전문, 인증정보가 로그에 없다.
- [ ] release에서 cleartext HTTP가 차단된다.
- [ ] `.\gradlew.bat test` 통과
- [ ] `.\gradlew.bat lint` 통과
- [ ] `.\gradlew.bat assembleDebug` 통과
